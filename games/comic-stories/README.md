# Comic Stories — new game module

A page-flipping comic reader that plugs into `baby_games` using the existing
`GameModule` / `GameRegistry` pattern — same shape as `alphabet-learner`.

It ships with one working demo story (placeholder art) so you can see it
running before you drop in your own comics.

## 1. Add the files to your repo

Copy this whole `comic-stories` folder into your repo at:

```
baby_games/games/comic-stories/
```

So you end up with:

```
games/
  comic-stories/
    ComicStoryGame.js
    styles.css
    assets/
      manifest.json
      images/
        demo-story/
          cover.svg, page-1.svg, page-2.svg, page-3.svg
```

## 2. Wire it into the platform (2 small edits)

**`index.html`** — add the stylesheet next to the alphabet-learner one:

```html
<link rel="stylesheet" href="games/alphabet-learner/styles.css">
<link rel="stylesheet" href="games/comic-stories/styles.css">
```

**`js/main.js`** — import and register it:

```js
import { ComicStoryGame } from '../games/comic-stories/ComicStoryGame.js';

// inside registerGames():
gameRegistry.register(ComicStoryGame);
```

That's it — "📖 Comic Stories" will now show up as a card on the launcher,
same as your alphabet game. The parent PIN (1234) already works everywhere
since it reuses the platform's `TimerService`.

Test locally first:

```
cd baby_games
python -m http.server 8000
```

then open `http://localhost:8000`.

## 3. Add your own stories

Each story is a folder of images plus one entry in `assets/manifest.json`.

1. Create a folder per story under `assets/images/`, e.g. `assets/images/goodnight-fox/`
2. Drop your comic panel images in there: `page-1.png`, `page-2.png`, etc., plus
   a `cover.png` (can just reuse page 1 if you don't want a separate cover)
3. Add an entry to `manifest.json`:

```json
{
  "id": "goodnight-fox",
  "title": "Goodnight, Little Fox",
  "coverImage": "goodnight-fox/cover.png",
  "pages": [
    { "image": "goodnight-fox/page-1.png", "caption": "One line of the story for this panel." },
    { "image": "goodnight-fox/page-2.png", "caption": "..." }
  ]
}
```

Notes:
- `caption` is optional but recommended — it's shown under the panel and is
  what gets read aloud by the "🔊 Read to me" button.
- Any image format works (PNG/JPG/WebP recommended; the demo uses SVG just
  because it's generated by code). No fixed aspect ratio is required — panels
  are scaled to fit, but a consistent ratio across a story looks tidiest
  (the demo uses 4:3 / 800×600).
- Add as many stories as you like; each becomes its own card in the library.

## 4. Turning your stories into comic panels

A few things that make AI-generated panels look like a coherent comic instead
of unrelated pictures:

- **Break the story into beats, not sentences.** One panel per key moment
  (roughly 1–2 sentences of narration each) reads much better than one panel
  per line of text.
- **Lock down a "character sheet" description once**, then reuse the exact
  same wording in every panel's prompt — e.g. *"a small round fox with orange
  fur, big white-tipped tail, wearing a blue scarf, flat cartoon style, bold
  black outlines, warm color palette"* — and just change the action/setting
  per panel. Consistent wording is what keeps the character looking the same
  across images, since each generation is independent.
- **Name the style once and repeat it**: pick something like *"children's
  storybook comic, flat colors, thick outlines, halftone shading"* and append
  it to every panel prompt.
- **Keep backgrounds simple** for a toddler audience — one clear focal action
  per panel reads better than a busy scene.
- Generate at a consistent aspect ratio (e.g. 4:3) so the reader doesn't
  jump around in size from panel to panel.

You already have a workflow for generating on-brand marketing images for
Hanana's soap business — the same "fixed character description + fixed style
phrase, vary the action" approach applies here.

## 5. Optional next steps

- Swap the demo's placeholder SVGs for your first real story, remove the
  `demo-story` entry from the manifest once you're happy.
- If a story gets long, consider a soft cap (8–12 panels) so a session fits
  comfortably inside the timer duration you set at the start.
